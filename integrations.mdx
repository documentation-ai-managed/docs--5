## Git Repository Syncing

Sync your documentation with GitHub or GitLab repositories to keep content up-to-date automatically. You configure a repository URL, and 孙 浩 pulls changes on a schedule or via webhooks.

<Steps>
  <Step title="Connect Repository" icon="git-branch">
    Navigate to Settings > Integrations > Git Sync.

    Enter your repository URL, such as `https://github.com/your-org/your-docs.git`.

    Provide a personal access token for authentication.
  </Step>
  <Step title="Configure Sync" icon="settings">
    Choose sync frequency: every `{5}` minutes, hourly, or on push.

    Select branches to monitor, like `main` and `develop`.
  </Step>
  <Step title="Test Sync" icon="play">
````bash
# Trigger manual sync via CLI (optional)
sunhao sync --repo https://github.com/your-org/your-docs.git --token YOUR_TOKEN
````
  </Step>
</Steps>

<Callout kind="tip">
Enable branch protection in your repo to prevent direct pushes—use pull requests for safer syncing.
</Callout>

## Webhook Setup for Notifications

Set up webhooks to notify external services on events like page updates or new versions.

<CodeGroup tabs="cURL,JavaScript">
````bash
curl -X POST https://api.example.com/v1/webhooks \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://your-webhook-url.com/webhook",
    "events": ["page.updated", "space.published"]
  }'
````

````javascript
const response = await fetch('https://api.example.com/v1/webhooks', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    url: 'https://your-webhook-url.com/webhook',
    events: ['page.updated', 'space.published']
  })
});
````
</CodeGroup>

## API Access for Custom Extensions

Build custom tools using the 孙 浩 API. Start by generating an API key in your account settings.

<ParamField path="spaceId" param-type="string" required="true">
  Your documentation space identifier.
</ParamField>

<ParamField header="Authorization" param-type="string" required="true">
  Bearer token: `Bearer YOUR_API_KEY`.
</ParamField>

<Request tabs="JavaScript,cURL">
````javascript
const response = await fetch('https://api.example.com/v1/spaces/{spaceId}/pages', {
  method: 'POST',
  headers: { 'Authorization': 'Bearer YOUR_API_KEY' },
  body: JSON.stringify({ title: 'New Page', content: '# Hello' })
});
````

````bash
curl -X POST https://api.example.com/v1/spaces/{spaceId}/pages \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"title": "New Page", "content": "# Hello"}'
````
</Request>

<Response tabs="200">
````json
{
  "id": "page-123",
  "title": "New Page",
  "status": "published"
}
````
</Response>
